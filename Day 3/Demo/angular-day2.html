🎨 Style Binding

Style binding lets you set inline CSS styles dynamically based on component data. You bind to specific style properties using [style.property].
Why? Sometimes you need to change colors, sizes, or visibility based on data — like showing a red price when it's expensive, or changing a progress bar width dynamically.

Single Style Binding

<!-- Bind a single style property -->
<p [style.color]="textColor">
  Colored Text
</p>

<!-- With units -->
<div [style.font-size.px]="fontSize">
  Dynamic Size
</div>

<div [style.width.%]="progress">
  Progress Bar
</div>

<!-- Conditional style -->
<p [style.color]="isUrgent ? 'red' : 'green'">
  {{ taskName }}
</p>

ngStyle – Multiple Styles

<!-- Multiple styles at once -->
<div [ngStyle]="{
  'color': textColor,
  'font-size': fontSize + 'px',
  'background-color': bgColor,
  'padding': '10px'
}">
  Styled Element
</div>

Component

export class TaskComponent {
  textColor = '#06b6d4';
  fontSize = 18;
  bgColor = '#1a1d2e';
  progress = 65;
  isUrgent = true;
  taskName = 'Fix login bug';
}
When to use which: Use [style.property] for a single style. Use [ngStyle] when you need to set multiple styles at once based on conditions.

🏷️ Class Binding

Class binding lets you add or remove CSS classes dynamically based on conditions. This is the Angular way to toggle classes instead of manual DOM manipulation.
Why? Toggling classes is the most common way to change UI appearance — active tabs, selected items, error states, completed tasks, etc.

Single Class Toggle

<!-- Add 'active' class when isActive is true -->
<div [class.active]="isActive">
  Tab Content
</div>

<!-- Toggle error class -->
<input [class.is-invalid]="hasError">

<!-- Completed task (strikethrough) -->
<li [class.completed]="task.done">
  {{ task.name }}
</li>

CSS

/* task.component.css */
.completed {
  text-decoration: line-through;
  opacity: 0.5;
}
.active {
  background: #7c3aed;
  color: white;
}
.is-invalid {
  border: 2px solid red;
}

ngClass – Multiple Classes

<!-- Object syntax -->
<div [ngClass]="{
  'active': isActive,
  'disabled': isDisabled,
  'highlight': isSelected
}">
  Multi-class Element
</div>

<!-- With task priority -->
<span [ngClass]="{
  'badge-high': task.priority === 'high',
  'badge-medium': task.priority === 'medium',
  'badge-low': task.priority === 'low'
}">
  {{ task.priority }}
</span>

Component

export class TaskComponent {
  isActive = true;
  isDisabled = false;
  isSelected = true;
  hasError = false;
  task = {
    name: 'Learn Angular',
    done: false,
    priority: 'high'
  };
}

Pattern: [class.name]="condition" for one class. [ngClass]="object" for multiple. Both are reactive — when the condition changes, Angular updates the DOM automatically.

🖱️ Event Binding ( )

Event binding lets you listen to user actions (clicks, key presses, mouse events) and call component methods. It uses parentheses ( ) syntax.
Why? Without events, your app is static. Event binding is how users interact with your application — clicking buttons, typing in inputs, submitting forms.

Event Binding Syntax

<!-- Click event -->
<button (click)="onClick()">
  Click Me
</button>

<!-- Pass data with $event -->
<input (keyup)="onKeyUp($event)">

<!-- Specific key -->
<input (keyup.enter)="onEnter()">

<!-- Mouse events -->
<div (mouseenter)="onHover()"
     (mouseleave)="onLeave()">
  Hover me
</div>

<!-- Input change -->
<select (change)="onFilter($event)">
  <option value="all">All</option>
  <option value="done">Done</option>
</select>

Component Methods

export class TaskComponent {
  message = '';

  onClick(): void {
    console.log('Button clicked!');
  }

  onKeyUp(event: KeyboardEvent): void {
    const input = event.target
      as HTMLInputElement;
    this.message = input.value;
  }

  onEnter(): void {
    console.log('Enter pressed!');
  }

  onFilter(event: Event): void {
    const select = event.target
      as HTMLSelectElement;
    console.log(select.value);
  }
}

Common Events

(click)Mouse click
(dblclick)Double click
(keyup)Key released
(keyup.enter)Enter key only
(input)Input value changes
(change)Select/checkbox changes
(focus) / (blur)Focus in/out

🖱️ Event Binding – Template Reference Variables

Instead of passing $event and casting it, you can use a template reference variable (#ref) to access the element directly. This is cleaner and more readable.

With $event (Works but verbose)

<input (keyup.enter)="addTask($event)">

// Component
addTask(event: KeyboardEvent): void {
  const input = event.target
    as HTMLInputElement;
  console.log(input.value);
  input.value = '';  // clear
}

With #ref (Cleaner)

<input #taskInput
  (keyup.enter)="addTask(taskInput.value);
    taskInput.value = ''">

// Component
addTask(name: string): void {
  console.log(name);
  // No casting needed!
}

Button + Input combo

<input #taskInput type="text"
  placeholder="Enter task name">

<button (click)="addTask(taskInput.value);
  taskInput.value = ''">
  Add Task
</button>
Key Point:
  • #taskInput gives you a reference to the actual <input> DOM element
  • taskInput.value gets the current text
  • You can pass it to any method as a parameter
  • No need for $event and type casting

🔄 Two-Way Binding [( )]

Two-way binding keeps the component property and the input value in sync — when the user types, the property updates. When the property changes in code, the input updates. Uses [(ngModel)].
Why? Without two-way binding, you'd need to manually listen to input events AND set input values. [(ngModel)] does both in one line — saving code and preventing bugs.

Setup – Import FormsModule

// app.module.ts
import { FormsModule }
  from '@angular/forms';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    FormsModule  // ← ADD THIS
  ],
  bootstrap: [AppComponent]
})
export class AppModule { }

Common Error: If you forget to import FormsModule, you'll get: Can't bind to 'ngModel' since it isn't a known property

Usage

<!-- Two-way binding -->
<input [(ngModel)]="taskName"
  placeholder="Enter task">

<!-- Live preview -->
<p>You typed: {{ taskName }}</p>

<!-- Works with select too -->
<select [(ngModel)]="priority">
  <option value="high">High</option>
  <option value="medium">Medium</option>
  <option value="low">Low</option>
</select>
<p>Priority: {{ priority }}</p>

<!-- Checkbox -->
<input type="checkbox" [(ngModel)]="isDone">
<p>Done: {{ isDone }}</p>
// Component
export class TaskComponent {
  taskName = '';
  priority = 'medium';
  isDone = false;
}
Banana in a Box: Remember [( )] as a banana 🍌 inside a box 📦. [ ] = property (data in), ( ) = event (data out), [( )] = both ways!

📊 All Binding Types – Summary

Angular has 4 types of data binding. Together they form the complete communication system between your component class and the template.
TypeSyntaxDirectionExample
Interpolation{{ }}Component → View<p>{{ title }}</p>
Property Binding[ ]Component → View<img [src]="url">
Event Binding( )View → Component<button (click)="save()">
Two-Way Binding[( )]Both ↔<input [(ngModel)]="name">

Sub-types of Property Binding

Sub-typeSyntax
DOM Property[src]="url"
Attribute[attr.colspan]="n"
Class[class.active]="bool"
Style[style.color]="val"
Multi-class[ngClass]="obj"
Multi-style[ngStyle]="obj"

Memory Trick:

  • {{ }} → Text content (display data)
  • [ ] → Data goes IN to the element
  • ( ) → Data comes OUT of the element
  • [( )] → Data goes BOTH ways

🔀 *ngIf – Conditional Rendering

*ngIf is a structural directive that adds or removes an element from the DOM based on a condition. If the condition is false, the element is completely removed — not just hidden.
Why? You often need to show/hide elements based on state — "show this message only if the list is empty", "show the edit form only when editing", "show a loading indicator while fetching data".

Basic *ngIf

<!-- Show only when true -->
<p *ngIf="isLoggedIn">
  Welcome back!
</p>

<!-- Show when list is empty -->
<p *ngIf="tasks.length === 0">
  No tasks yet. Add one!
</p>

<!-- Show only if task exists -->
<div *ngIf="selectedTask">
  <h3>{{ selectedTask.name }}</h3>
  <p>{{ selectedTask.description }}</p>
</div>

*ngIf with else

<!-- if/else pattern -->
<div *ngIf="tasks.length > 0;
          else emptyState">
  <ul>
    <li *ngFor="let t of tasks">
      {{ t.name }}
    </li>
  </ul>
</div>

<ng-template #emptyState>
  <p>No tasks yet!</p>
</ng-template>

Component

export class TaskListComponent {
  isLoggedIn = true;
  tasks: any[] = [];
  selectedTask: any = null;
  isEditing = false;

  toggleEdit(): void {
    this.isEditing = !this.isEditing;
  }
}

*ngIf vs [hidden]

Feature*ngIf[hidden]
DOMRemoves elementHides with CSS
PerformanceBetter for heavy elementsBetter for frequent toggles
LifecycleDestroys & recreatesStays alive
Rule of thumb: Use *ngIf by default. Use [hidden] only when you need the component to stay alive (e.g., preserve form data).

🔧 Built-in Pipes – Data Formatting

A Pipe transforms data in the template before displaying it. Think of it as a filter — the data goes in, gets formatted, and comes out. Syntax: {{ value | pipeName }}.
Why? You often need to format dates, currencies, and text for display. Pipes let you do this in the template without changing the original data in the component.

Text Pipes

<!-- uppercase -->
{{ 'hello' | uppercase }}
<!-- Output: HELLO -->

<!-- lowercase -->
{{ 'HELLO' | lowercase }}
<!-- Output: hello -->

<!-- titlecase -->
{{ 'fix login bug' | titlecase }}
<!-- Output: Fix Login Bug -->

Date Pipe

<!-- Default -->
{{ today | date }}
<!-- Mar 17, 2026 -->

<!-- Short date -->
{{ today | date:'shortDate' }}
<!-- 3/17/26 -->

<!-- Custom format -->
{{ today | date:'dd/MM/yyyy' }}
<!-- 17/03/2026 -->

<!-- Full date -->
{{ today | date:'fullDate' }}
<!-- Tuesday, March 17, 2026 -->

Number & Currency Pipes

<!-- Currency -->
{{ 1999 | currency }}
<!-- $1,999.00 -->

{{ 1999 | currency:'EGP' }}
<!-- EGP1,999.00 -->

<!-- Number formatting -->
{{ 3.14159 | number:'1.0-2' }}
<!-- 3.14 -->

<!-- Percent -->
{{ 0.85 | percent }}
<!-- 85% -->

JSON Pipe (Debugging)

<!-- Great for debugging objects -->
<pre>{{ task | json }}</pre>
<!-- {
  "name": "Learn Angular",
  "done": false,
  "priority": "high"
} -->

Chaining Pipes

<!-- Chain multiple pipes -->
{{ task.name | slice:0:10 | uppercase }}
<!-- LEARN ANGU -->
Component: today = new Date(); — The date pipe needs a Date object or a date string.
